# Observation: TC-0005 (content selection variance, JSON summary)

## Summary

TC-0005 was designed to test:

- wording / content selection variance
- metadata-driven evaluation behavior

The case enforces:

- JSON output
- exactly 2 sentences in summary
- multiple important information points

This reliably produces normalized diffs across runs.

---

## Compare Layer Behavior

For all tested configurations:

- formatMatch = true
- rawDiffDetected = true
- normalizedDiffDetected = true

severityHint varies depending on policy:

- low-drift → HIGH
- flexible → MEDIUM

This confirms:

- compare-run correctly detects both raw and normalized differences
- severityHint is influenced by changePolicy

---

## Evaluation Layer Behavior

### Case 1: assertionMode = loose, changePolicy = low-drift

- recommendedVerdict: FAIL
- reasons:
  - normalized diff detected
  - low-drift policy escalated normalized diff

Interpretation:
- low-drift treats normalized differences as unacceptable drift

---

### Case 2: assertionMode = strict, changePolicy = low-drift

- recommendedVerdict: FAIL
- reasons:
  - normalized diff detected
  - strict assertion mode requires exact stability
  - low-drift policy escalated normalized diff

Interpretation:
- strict mode adds an explicit requirement for exact stability
- same diff is treated more strictly than loose mode

---

### Case 3: assertionMode = loose, changePolicy = flexible

- recommendedVerdict: REVIEW
- reasons:
  - normalized diff detected

Interpretation:
- flexible policy allows normalized differences without escalation to FAIL
- REVIEW is used instead of FAIL

---

## Key Findings

1. assertionMode is active in evaluation
   - strict mode introduces stronger reasoning requirements
   - reflected in "reasons" field

2. changePolicy is active in evaluation
   - low-drift escalates normalized diffs → FAIL
   - flexible keeps normalized diffs → REVIEW

3. compare-run and eval-run separation is preserved
   - compare-run provides evidence only
   - eval-run interprets evidence using metadata

---

## Design Implications

- metadata-driven evaluation is functioning as intended
- policy-based recommendation is already effective in Phase 7A
- no immediate need for core redesign

---

## Future Considerations (Phase 7B)

- refine how normalizedDiffDetected is interpreted
- introduce gradation of diff severity (minor vs major drift)
- improve semantic comparison (meaning preservation vs wording change)
