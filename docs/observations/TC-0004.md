# Observation: TC-0004 (wording variance case)

## Summary

When using TC-0004 with non-empty responses:

- rawDiffDetected = true
- normalizedDiffDetected = true
- severityHint = MEDIUM
- recommendedVerdict = REVIEW

This confirms that:

- compare-run correctly detects content differences
- eval-run produces REVIEW for non-trivial differences

## Additional finding

Even when the response was a JSON object:

{
  "summary": "..."
}

formatMatch was still reported as true under:

- expectedFormat: "text"

## Interpretation

Current "text" format validation appears to be:

→ lenient (any non-empty string is treated as valid text)

It does NOT strictly enforce:

- "plain text only"
- "no structured formats like JSON"

## Status

- Working as implemented
- Not considered a bug
- Identified as a design gap for future refinement
