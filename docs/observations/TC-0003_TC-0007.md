# Phase 7B Design Input
## Based on Trial & Expansion (TC-0003 ~ TC-0007)

---

# 1. Summary

The current framework (Phase 7A) successfully implements:

- evidence-based comparison (compare-run)
- metadata-driven recommendation (eval-run)
- strict separation between evidence and judgment

Trial cases TC-0003 to TC-0007 confirm that:

- format violations are strongly detected
- content differences are consistently captured
- metadata (assertionMode, changePolicy) influences recommendation
- omission-like differences can be escalated via policy

---

# 2. Observed Strengths

## 2.1 Format Validation is Strong

From TC-0003, TC-0007:

- formatMatch = false leads to immediate FAIL
- format mismatch overrides content comparison

→ Format contract enforcement is reliable

---

## 2.2 Diff Detection is Stable

Across all cases:

- rawDiffDetected and normalizedDiffDetected are consistently triggered
- compare-run produces clear evidence signals

→ Diff detection layer is robust

---

## 2.3 Metadata-driven Evaluation Works

From TC-0005:

- assertionMode = strict introduces stronger reasoning
- changePolicy = low-drift escalates to FAIL
- changePolicy = flexible allows REVIEW

→ Metadata is actively used in evaluation

---

## 2.4 Omission Can Be Escalated

From TC-0006:

- omission-like differences result in FAIL under:
  - high priority
  - low-drift policy

→ Even without explicit semantic understanding, policy-based escalation is effective

---

# 3. Observed Limitations

## 3.1 No Explicit Semantic Understanding

Current evaluation treats:

- wording changes
- content shifts
- omission

all as:

→ "normalized diff"

There is no distinction between:

- minor paraphrasing
- important information loss

---

## 3.2 Omission is Not Explicitly Modeled

From TC-0006:

- missing critical information is not identified as "omission"
- instead treated as generic diff

→ Important semantic errors are not labeled

---

## 3.3 Diff Severity is Flat

normalizedDiffDetected is binary:

- true / false

But real-world differences have gradation:

- trivial wording change
- partial information loss
- critical omission

→ No severity spectrum beyond HIGH/MEDIUM hints

---

## 3.4 Format vs Meaning Priority is Rigid

From TC-0007:

- format mismatch always dominates evaluation

This is correct in most cases, but:

- mixed cases (format + semantic issue) are not decomposed

---

## 3.5 Case-level Constraints Are Not Evaluated

Example:

- "must include customer support handling"

This requirement is:

- not explicitly checked
- only indirectly reflected via diff

---

# 4. Key Insight

Current system evaluates:

→ "difference existence + policy"

But does NOT evaluate:

→ "difference meaning"

---

# 5. Phase 7B Design Directions

## 5.1 Introduce Semantic Signals

Add new evidence fields (compare layer):

- possibleOmissionDetected
- keyInfoMissing (heuristic)
- structuralDeviationType

These remain evidence-only.

---

## 5.2 Distinguish Diff Types

Instead of single normalizedDiff:

- minorDiff (wording)
- structuralDiff (sentence structure)
- semanticDiff (meaning change)
- omissionDiff (missing required info)

---

## 5.3 Improve Evaluation Reasoning

eval-run should:

- interpret diff types differently
- produce richer reasons

Example:

- "critical information missing"
- "semantic meaning changed"
- "minor paraphrasing detected"

---

## 5.4 Policy × Diff-Type Matrix

Introduce interaction model:

| Diff Type     | low-drift | flexible |
|---------------|----------|----------|
| minor         | REVIEW   | PASS     |
| semantic      | FAIL     | REVIEW   |
| omission      | FAIL     | FAIL/REVIEW |
| format        | FAIL     | FAIL     |

---

## 5.5 Optional Case-level Assertions

Future extension:

- mustInclude: ["customer support"]
- mustMention: ["release delay"]

These can be used as hints for omission detection.

---

# 6. Migration Strategy

Important:

- Do NOT break current Phase 7A behavior
- Add new signals incrementally
- Keep compare-run as evidence-only
- Keep eval-run as interpretation layer

---

# 7. Status

- Phase 7A validated through TC-0003 ~ TC-0007
- Core architecture is stable
- Ready to begin Phase 7B (Smarter Evaluation)
